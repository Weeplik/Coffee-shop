# Техническое задание на верстку одностраничного сайта кофейни «Coffee Shop»

## 1. Характеристика предметной области

Предметной областью является сфера общественного питания, а именно деятельность кофейни. Кофейня предоставляет посетителям информацию о меню, специальных предложениях, режиме работы и контактных данных через веб-сайт.

Разрабатываемая страница предназначена для представления кофейни в сети Интернет и привлечения потенциальных клиентов.

---

## 2. Тема разработки

Верстка одностраничного сайта (Landing Page) для кофейни «Coffee Shop».

---

## 3. Назначение разрабатываемой страницы

Разрабатываемая веб-страница предназначена для:

* представления информации о кофейне;
* демонстрации ассортимента напитков и десертов;
* информирования посетителей о преимуществах заведения;
* предоставления контактной информации;
* привлечения новых клиентов.

---

## 4. Функциональные требования

### 4.1 Шапка сайта (Header)

Страница должна содержать:

* логотип кофейни;
* навигационное меню;
* кнопку перехода к разделу контактов.

### 4.2 Главный экран (Hero Section)

Страница должна содержать:

* фоновое изображение;
* название кофейни;
* краткое описание;
* кнопку «Заказать столик».

### 4.3 Блок преимуществ

Отображение не менее трех преимуществ кофейни:

* качественный кофе;
* свежая выпечка;
* уютная атмосфера.

### 4.4 Блок меню

Отображение карточек товаров:

* изображение;
* название;
* описание;
* цена.

Количество карточек — не менее 6.

### 4.5 Блок «О нас»

Должен содержать:

* краткую информацию о кофейне;
* историю создания;
* миссию заведения.

### 4.6 Блок контактов

Должен содержать:

* адрес;
* номер телефона;
* адрес электронной почты;
* карту расположения (по желанию).

### 4.7 Подвал сайта (Footer)

Должен содержать:

* информацию об авторских правах;
* ссылки на социальные сети.

---

## 5. Нефункциональные требования

### 5.1 Требования к используемым языкам программирования

Для разработки необходимо использовать:

* HTML5;
* CSS3;
* JavaScript (ES6).

Использование серверных технологий не требуется.

### 5.2 Требования к дизайну

#### Цветовая гамма

* основной цвет: `#6F4E37`;
* дополнительный цвет: `#D2B48C`;
* цвет текста: `#333333`;
* цвет фона: `#FFFFFF`.

#### Шрифты

* заголовки — Montserrat;
* основной текст — Open Sans.

#### Требования к интерфейсу

* современный минималистичный стиль;
* единое оформление всех блоков;
* адаптивность для мобильных устройств;
* удобная навигация.

### 5.3 Требования к стилю написания кода

* использование семантических тегов HTML5;
* соблюдение единого стиля оформления кода;
* именование CSS-классов по методологии БЭМ;
* отсутствие дублирования кода;
* наличие комментариев для сложных участков кода;
* корректные отступы и форматирование.

---

## 6. Требования к документации
